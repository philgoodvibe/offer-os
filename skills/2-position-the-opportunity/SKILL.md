# Step 2 — Position the Opportunity

> What the customer *feels* (Step 1's root problem) and what the *category* keeps doing wrong are the same defect seen from two sides. Find the shortcoming every alternative shares, build the option that removes it, and you've created a category instead of competing in one.

## First, classify the angle
- **Fix-it** positioning sells to *need* — necessary, but unexciting.
- **Improvement** positioning sells to *obligation* — weak; you can spot it by **"-er" words** (better / faster / cheaper / easier). Most businesses are quietly here and don't realize it.
- **New-category** positioning sells to *desire* — the only one that drives action and supports premium pricing.

## The switch (6 moves)
1. **Lock the destination.** The result the customer wants never changes — only the path does. Write it in one sentence and leave it alone.
2. **Map the alternatives.** List every option they've tried or considered — rival methods, doing nothing, doing something completely different. Aim for 10+. The fuller the map, the stronger the move.
3. **Find the shared shortcoming.** Ask one question of each alternative: *"why did this fail them?"* One answer repeats — not a flaw of one option, a defect built into the **whole category.** Said plainly to the customer, that shortcoming removes their self-blame.
4. **Design what the shortcoming forbids.** Don't brainstorm — invert. Define your option by what it does NOT require. The path should carry the work (not the customer's willpower), and feel familiar enough to trust yet different enough to believe *"this could work where the others didn't."*
5. **Name the mechanism, name the identity.** People buy a named *thing*, not a process. Give the mechanism a noun and give the customer an identity they can claim on day one. Then write the line: *"I help [specific sub-market] [reach the destination] with [the mechanism]."*
6. **Run the five checks** (pass all or return to move 4):
   - **Different, not better** — no "-er" claims; a genuinely different path.
   - **Wrong shelf** — it shouldn't sit beside the things they already quit.
   - **Removes blame** — "it wasn't you; the old way was built wrong."
   - **Carries the load** — the mechanism does the work, not their discipline.
   - **Status on day one** — claimable immediately, and priceable above the category.

## Worked example (illustrative)
*The shift-worker meal service.* Destination: "Eat well without spending my off-hours cooking." Alternatives: meal-kit boxes, generic delivery apps, weekend batch-cooking, protein bars, "just eat better," doing nothing. Shared shortcoming: **every option still assumes you have energy and a predictable schedule to cook or plan** — exactly what a rotating-shift worker doesn't have. Invert it: pre-made, shift-timed meals that need zero planning and zero cooking. Mechanism: "shift-synced meals." Identity: "a fueled shift worker." Five checks: it's a different path (not "easier cooking"), wrong shelf (not a meal kit), removes blame (your schedule broke the old options, not your willpower), carries the load (no planning required), status day one. 

## BUILD mode
1. Pull `destination` + `competing_vehicles`; expand the alternatives map to 10+.
2. Find the one shortcoming all alternatives share; phrase it as the customer's absolution.
3. Invert it into your option; name the mechanism (noun) + identity; write the who/what line.
4. Run the five checks; iterate until all pass.
5. **Output:** positioning statement, mechanism/identity names, who/what line. Save; hand the named mechanism to **Step 3 (frameworks)** and **Step 4 (offer)**.

### Copy-paste AI prompt
```
Position this as a new category. Inputs: <destination + competing_vehicles + the root problem from Step 1>.
1) Lock the destination in one sentence.
2) Map 10+ alternatives (incl. "do nothing" and "do something different").
3) Ask "why did this fail them?" of each; name the ONE shortcoming the whole category shares.
4) Invert it: describe the option defined by what it does NOT require; the path carries the work.
5) Name the mechanism (a noun) and the identity (claimable day one); write "I help [sub-market] [destination] with [mechanism]."
6) Score against the 5 checks (different/wrong-shelf/removes-blame/carries-load/status), 0-10 each; fix anything < 8.
```

## AUDIT mode — rubric (used by /marketing-360)
Score current positioning 0–10 = the five checks, 2 points each.
🔴 0–4 (it's an *improvement* — "-er" words, same aisle) · 🟡 5–7 (differentiated, not yet a category) · 🟢 8–10 (a new category priced above the field).
