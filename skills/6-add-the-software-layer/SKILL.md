# Step 6 — Add the Software Layer

> Information alone is getting harder to sell — buyers can extract a lot of it from an AI for free. Software alone is commoditized — anyone can generate it. The edge is the combination: **sell the information, include software that runs it as the bonus.** The information drives the transformation; the tool makes the offer hard to copy and easy to use.

## Why now
Modern AI tools collapse the old "I can't build software" barrier — a single step of a framework can become a working tool in an afternoon. A small tool attached to an info offer routinely lifts perceived value and stickiness far more than its build cost.

## What to build
A **small, single-purpose tool** that automates ONE step of your framework. Keep it tiny:
- as a **free lead magnet** (a one-job tool behind a curiosity hook — pairs with Step 5 lead gen), or
- as a **paid-offer bonus** (the "it's basically done for you" lever from Step 4/5).

## BUILD mode
1. Take the flagship framework (Step 3); find the single most tedious or error-prone step.
2. Propose 3 single-purpose tools that automate that step. For each: the **one job**, the **curiosity hook**, lead-magnet vs paid-bonus, and a **one-paragraph build spec** (inputs → logic → output).
3. Pick one; note where it plugs into the funnel.
4. **Output:** tool concepts + specs. Save; hand the chosen tool to **Step 7 (launch)** as a magnet or bonus.

```

Keep each tool to ONE job. No platforms, no suites.
```

## AUDIT mode — rubric (used by /marketing-360)
0–10: **a tool is attached** (or a concrete plan) (4) · it's **single-purpose & narrow** (not a bloated platform) (3) · positioned as the **bonus**, with information as the product (3).
🔴 0–4 (pure info a buyer can now get free from an AI) · 🟡 5–7 (a tool, but bloated or mis-positioned) · 🟢 8–10 (a tight tool that makes the info offer hard to copy).
