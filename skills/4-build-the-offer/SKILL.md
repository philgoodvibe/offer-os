# Step 4 — Build the Offer

> An offer sells on what the buyer feels and when — not on how big the bundle is. Two moves: assemble the right contents from the buyer's problem-path, then reveal them in the order the buyer hits each problem.

## Move 1 — Assemble from the problem-path
1. Define the **end state** the buyer wants.
2. List **every** obstacle between where they are and that end state.
3. Attach **exactly one** solution to each obstacle. One problem → one component.

### Cut bloat — the relevance filter
A component belongs only if it moves the buyer along the path. Reject by three rules:
- *it's valuable* ≠ it belongs · *you spent time making it* ≠ it belongs · *it sells well elsewhere* ≠ it belongs.
Not everything has to come from you — partner or license where it serves the path.

## Move 2 — Sequence the reveal
Present components **one at a time, in the order the buyer hits each obstacle.** As each new objection surfaces, answer it with the next component:
> "Once you have **[component 1]**, you might be thinking *[next objection]* — that's exactly what **[component 2]** is for."
The buyer pictures themselves reaching the finish line. When they can see the win, they find a way to buy.

## Move 3 — Stack and price
Always build a stack for two reasons: to make the offer **un-comparable** and to make the value **obvious**. Line-item each component with a dollar value → total it → reveal the price well below the total.

> It's architecture, not content: the same product, re-mapped and re-sequenced, routinely multiplies conversion and profit without changing what's delivered.

## BUILD mode
1. Pull `destination` + avatar; use the flagship framework (Step 3) as the spine.
2. Map the problem-path; attach one solution per obstacle; run the relevance filter.
3. Order the stack by problem-sequence; write the objection-relief line for each handoff.
4. Assign values; build the value/price slide.
5. **Output:** the lean stack, the slide, the script. Save; hand to **Step 5 (funnel)**.

## AUDIT mode — rubric (used by /marketing-360)
0–10: **one solution per real obstacle** (no bloat) (3) · components are **on-path** (2) · **sequenced** to relieve objections (2) · **value-stacked** with a clear price anchor (3).
🔴 0–4 (a random bundle / "more is better") · 🟡 5–7 (good components, weak sequence or pricing) · 🟢 8–10 (a path the buyer can see themselves walk).
