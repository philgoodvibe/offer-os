# Step 5 — Build the Funnel

> Pick the funnel type that matches your goal, then make the math work. **Core rule: average order value must exceed cost to acquire** — break even up front, profit as the customer moves up the ladder.

## Pick the type

### Lead generation
A free asset that gates a *single* high-curiosity promise: **one specific win, instant access.** Build it as a splinter of the core offer, a distilled high-value resource, or a tiny single-feature tool. Promise one thing only.

### Low-ticket
A small front-end purchase plus order bumps and a short upsell sequence (buyer → bump → upsell 1 → upsell 2 → thank-you). Aim for average order value above acquisition cost. **Unbundle** a larger offer across several pages, or for physical goods, present quantity tiers and let most buyers self-select more than one unit. For info products, sell the *next* step; for physical, sell *more of the same.*

### Continuity (recurring revenue)
A genuinely useful but intentionally incomplete free/cheap front end whose completion lives inside a membership. Feed it with a steady stream of fresh front-end hooks that all route to the same subscription.

### One-to-many
A webinar or multi-day challenge that sells to a room at once. (Run-of-show and post-pitch bonuses live in **Step 7**.)

### High-ticket (often the best place to start)
At high prices the margin forgives imperfect ads and copy, and buying itself raises the buyer's commitment. Sell inside immersive experiences. Write the offer as a flexible 1–2 page document with these elements:
1. **Outcome-and-speed name** ("Done-in-a-Day Intensive") · 2. **The named system** · 3. **A speed promise** · 4. **Layered support** (coaches/experts) · 5. **Experience** (in-person / events) · 6. **Exposure** (stage, audience, access) · 7. **Done-for-you element** (removes the last objection).

## BUILD mode
1. Read the goal + price ladder. Recommend the type (or sequence of types).
2. Lay out the funnel page-by-page; for high-ticket, draft the 1–2 page offer doc.
3. Compute the economics: acquisition-cost target, front-end average order value, upsell take-rates → confirm value beats cost.
4. **Output:** type + funnel map + math. Save; hand to **Step 6 (software)** and **Step 7 (launch)**.

## AUDIT mode — rubric (used by /marketing-360)
0–10: the **type fits the goal** (3) · the **structure is complete** (bumps/upsells or the high-ticket elements) (3) · the **math works** (value beats acquisition cost) (4).
🔴 0–4 (one page, no upsells, no math) · 🟡 5–7 (structure ok, economics unproven) · 🟢 8–10 (right type, full structure, profitable math).
