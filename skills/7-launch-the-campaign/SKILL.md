# Step 7 — Launch the Campaign

> This is where the assembly line ships. Run it **one-to-many** (a webinar or a multi-day challenge), and design for a truth most launches miss: **a large share of sales comes after the pitch, not during it.**

## The follow-on bonus structure
Make the pitch a midpoint, not the finale. After the offer goes live, keep teaching: each remaining session introduces a new idea, unlocks a **bonus tied to that idea**, and redirects back to the offer. The stack visibly grows day over day, which keeps the room engaged and pulls in buyers who weren't ready on pitch day. The single-purpose tool from Step 6 and partner tools make strong follow-on bonuses. Even a one-shot webinar should add at least one bonus beat after the pitch.

## Assemble the campaign (orchestrator)
Chain the sequence into one brief — each step feeds the next:
1. **Problem** (Step 1) → the registration angle.
2. **Opportunity** (Step 2) → the new path the event reveals.
3. **Frameworks** (Step 3) → the teaching content.
4. **Offer** (Step 4) → the stack + value slide you pitch.
5. **Funnel** (Step 5) → registration → show → pitch → upsells.
6. **Software** (Step 6) → the magnet that fills the room and the bonus that closes it.
7. **Launch** (here) → the run-of-show + follow-on bonus days + the calendar.

## BUILD mode
1. Pull the outputs of Steps 1–6 (or run them first if missing).
2. Choose the mechanism (single webinar vs multi-day challenge) for the price point and audience.
3. Write the **day-by-day run-of-show**: pre-frame → teach → pitch → **follow-on bonus days** (one new idea + tied bonus per day) → close.
4. Produce a **launch calendar** (fill-the-room → live → cart-close) with the metric to watch in each phase.
5. **Output:** the campaign brief + run-of-show + calendar. Save the full brief via the adapter.

```

4) Output a one-page campaign brief that ties back to the Problem (Step 1) and the new path (Step 2).
```

## AUDIT mode — rubric (used by /marketing-360)
0–10: a real **one-to-many mechanism** (not post-and-pray) (3) · **post-pitch follow-on bonuses** planned (3) · a concrete **promo calendar** with phases + metrics (2) · it **ties back to the Problem + Opportunity** (2).
🔴 0–4 (no launch mechanism) · 🟡 5–7 (a webinar, but no follow-on engine or calendar) · 🟢 8–10 (a sequenced one-to-many launch with after-pitch revenue built in).
